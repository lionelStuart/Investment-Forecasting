# AI 投资理财系统预研分析

本文档整理项目早期预研结论，用于指导一期 MVP 的技术选型、数据接入、量化预测、MCP 服务设计和每日理财建议生成。

## 1. 项目定位

本项目目标不是让 AI 直接凭自然语言猜测行情，而是构建一套可复现、可回测、可审计的投资理财辅助系统。

推荐分工：

```text
数据与量化服务
  负责数据获取、清洗、指标计算、模型预测、回测评分

MCP 服务
  负责把数据和模型能力封装为结构化工具，供 AI 稳定调用

AI / Codex
  负责理解用户目标、调度工具、解释结果、生成每日建议和报告

WebUI
  负责展示数据、预测结果、每日建议、历史评分和任务日志
```

核心原则：

- AI 不直接“猜涨跌”，必须基于数据和模型输出整理建议。
- 预测必须使用结构化结果，而不是不可追踪的自然语言判断。
- 每条建议都需要有风险边界、关键假设和历史表现校验。
- 系统应持续记录预测结果和真实结果，用于评分和模型优化。

## 2. 数据源预研

### 2.1 AKShare

AKShare 是一期 MVP 推荐优先使用的数据源。

优点：

- 免费。
- Python 直接调用。
- 覆盖 A 股、指数、ETF、LOF、公募基金、宏观等数据。
- 适合快速原型和本地研究系统。

缺点：

- 很多接口来源于东方财富、同花顺、天天基金等网页或公开接口。
- 源站接口变化可能导致调用失败。
- 生产环境需要缓存、重试、数据校验和降级方案。

示例：

```python
import akshare as ak

# A 股历史行情
stock_df = ak.stock_zh_a_hist(
    symbol="600519",
    period="daily",
    start_date="20240101",
    end_date="20260522",
    adjust="qfq",
)

# 开放式基金每日净值概览
fund_df = ak.fund_open_fund_daily_em()

# 单只开放式基金历史净值
nav_df = ak.fund_open_fund_info_em(
    symbol="000001",
    indicator="单位净值走势",
)

# ETF 历史行情
etf_df = ak.fund_etf_hist_em(
    symbol="510300",
    period="daily",
    start_date="20240101",
    end_date="20260522",
    adjust="qfq",
)
```

适合 MVP 的数据范围：

- A 股日线。
- 指数日线。
- ETF / LOF 行情。
- 开放式基金净值。
- 公募基金基础信息。
- 偏股型基金筛选。

### 2.2 Tushare Pro

Tushare Pro 适合作为后续增强数据源。

优点：

- 接口结构更规范。
- 适合构建长期数据仓库。
- 覆盖 A 股行情、每日指标、财务数据、指数、基金、资金流等。

缺点：

- 需要注册 token。
- 部分接口需要积分或权限。
- 不适合作为零门槛 MVP 的唯一数据源。

示例：

```python
import tushare as ts

pro = ts.pro_api("your_token")

# A 股日线
daily_df = pro.daily(
    ts_code="600519.SH",
    start_date="20240101",
    end_date="20260522",
)

# 每日估值和换手等指标
basic_df = pro.daily_basic(
    ts_code="600519.SH",
    start_date="20240101",
    end_date="20260522",
    fields="ts_code,trade_date,close,turnover_rate,pe,pb,total_mv,circ_mv",
)
```

适合后续增强：

- 股票估值指标。
- 财务指标。
- 基金持仓。
- 指数成分。
- 融资融券。
- 资金流。

### 2.3 BaoStock

BaoStock 可作为免费 A 股历史行情补充源。

优点：

- 免费。
- 接口结构化。
- 适合获取 A 股历史 K 线和部分财务数据。

缺点：

- 基金覆盖较弱。
- 生态和活跃度不如 AKShare / Tushare。
- 更适合作为补充数据源，而不是主数据源。

### 2.4 天天基金 / 东方财富

天天基金和东方财富是基金净值、基金列表、基金阶段收益和展示字段的重要来源。

建议：

- MVP 中优先通过 AKShare 间接调用。
- 不建议直接把网页爬取作为核心数据能力。
- 如果后续需要补充字段，可以单独封装适配器并强制缓存。

## 3. 数据接入建议

一期数据层建议采用：

```text
AKShare + SQLite + 本地缓存 + 失败重试
```

后续增强为：

```text
AKShare + Tushare Pro + 宏观数据源 + SQLite/DuckDB/PostgreSQL
```

建议设计统一数据适配器：

```text
data_sources/
  akshare_provider.py
  tushare_provider.py
  baostock_provider.py
  cache.py
  schema.py
  update_daily.py
```

统一输出字段，避免上层模型直接依赖某个数据源的原始字段。

## 4. 量化预测实现思路

“准确度高”不能理解为保证预测涨跌，而应拆解为可度量目标：

- 未来 5 / 20 / 60 个交易日上涨概率。
- 未来 20 / 60 个交易日收益区间。
- 未来 20 / 60 个交易日最大回撤风险。
- 是否跑赢沪深 300、中证 500 或中证偏股基金指数。
- 当前市场状态适合激进、中等还是保守配置。

不要直接预测单点价格。更推荐预测：

```text
未来 60 日是否跑赢基准
未来 20 日最大回撤是否超过阈值
未来 20 日收益是否位于正收益区间
未来 60 日适合的权益仓位区间
```

### 4.1 推荐模型路线

MVP 优先：

- 基准模型：历史均值、动量、均线、指数基准。
- 机器学习模型：LightGBM / XGBoost / RandomForest。
- 风险模型：波动率、最大回撤、下行波动、相关性。
- 规则模型：市场状态、仓位上限、止损和再平衡规则。

后续增强：

- GARCH 风险建模。
- LSTM / Transformer 时间序列模型。
- NLP 新闻、公告、基金季报情绪分析。
- 多模型集成和模型投票。

实践建议：

- 先用简单模型建立可复现基线。
- 复杂模型必须证明样本外收益、回撤或稳定性优于简单基线。
- 不应只优化方向准确率，也要优化回撤控制和组合收益。

## 5. 历史回测与模型校准

开发期需要采集旧数据，用多段数据集评估模型。

核心方式：

```text
在时间点 T：
  只允许使用 T 之前的数据
  用模型预测 T + x 个交易日后的表现
  读取真实结果
  计算预测误差和建议评分
```

示例数据切分：

```text
样本 A：2018-01-01 至 2020-12-31
样本 B：2021-01-01 至 2022-12-31
样本 C：2023-01-01 至 2025-12-31
```

每段样本都应覆盖不同市场状态：

- 牛市。
- 熊市。
- 震荡市。
- 高波动期。
- 低波动期。
- 风格切换期。

必须避免：

- 未来函数。
- 随机切分时间序列。
- 未扣除交易成本。
- 只看收益不看回撤。
- 在全量历史数据上反复调参导致过拟合。

## 6. 预测建议评分

每条预测和建议都应被记录并评分。

评分指标：

```text
direction_accuracy
  方向是否判断正确

return_error
  预测收益和真实收益的偏差

risk_hit
  是否识别到主要回撤风险

benchmark_excess
  是否跑赢基准

drawdown_control
  回撤是否低于预期风险边界

advice_executability
  仓位、触发条件和风险提示是否清晰可执行
```

建议输出：

```json
{
  "prediction_score": 82,
  "risk_score": 76,
  "advice_score": 88,
  "overall_score": 81
}
```

评分结果需要持久化，用于：

- 模型版本对比。
- 策略参数优化。
- 每日建议质量追踪。
- WebUI 展示历史表现。

## 7. MCP 服务设计

MCP 应暴露结构化工具，让 AI 可以调用数据与量化能力。

建议一期工具：

```text
get_asset_list
  获取资产列表，支持股票、指数、ETF、基金、偏股型基金筛选

get_asset_history
  获取指定资产的历史行情或净值

get_fund_metrics
  获取基金收益、波动、回撤、夏普、同类排名等指标

get_market_snapshot
  获取当前市场状态摘要

run_forecast
  对资产、基金或组合执行预测

run_backtest
  对指定模型、资产、区间和预测周期执行回测

get_daily_advice
  获取某日理财建议

generate_daily_advice
  触发完整数据更新、预测、评估和建议生成
```

MCP 输出应使用 JSON：

```json
{
  "asset": "沪深300ETF",
  "horizon_days": 60,
  "prediction": {
    "expected_return": 0.045,
    "up_probability": 0.61,
    "downside_risk": -0.085,
    "confidence": 0.68
  },
  "backtest": {
    "annual_return": 0.128,
    "max_drawdown": -0.172,
    "sharpe": 1.05,
    "win_rate": 0.57
  },
  "risk_factors": [
    "估值分位偏高",
    "市场成交额下降",
    "宏观流动性边际收紧"
  ]
}
```

AI 基于 MCP 输出生成自然语言建议：

```text
未来 60 个交易日模型偏乐观，上涨概率约 61%，预期收益约 4.5%。
但最大回撤压力约 8.5%，且估值分位偏高，因此不建议一次性满仓。
适合分批配置，目标仓位 20%-30%，若市场宽度继续恶化则降低至 10%-15%。
```

## 8. 每日定时任务设计

需要建立 Codex 定时任务：

```text
每天 08:00 自动请求 Codex CLI
执行一次数据更新、量化分析、预测评估和每日理财建议生成
```

每日任务流程：

```text
1. 拉取最新行情、指数、基金净值和市场数据
2. 更新 SQLite
3. 执行数据质量检查
4. 计算最新特征和风险指标
5. 运行预测模型
6. 生成激进、中等、保守三类建议
7. 更新历史预测评分
8. 写入 daily_advice
9. 写入 task_logs
10. WebUI 展示最新结果
```

注意：中国市场在交易日收盘后才有完整日线和基金净值。早上 8 点生成的是开盘前指导建议，应主要基于上一交易日数据、隔夜信息和历史模型结果。

## 9. 每日建议形态

每日建议应分三类：

### 激进型

- 更高权益仓位。
- 更关注弹性资产和成长风格。
- 明确回撤容忍度和减仓条件。

### 中等型

- 权益、债券、现金类资产均衡配置。
- 倾向分批买入或定投。
- 根据市场状态动态调整仓位。

### 保守型

- 更重视流动性和回撤控制。
- 更高比例配置现金、货币基金、短债或低波动资产。
- 降低权益暴露。

建议不应只有“买/卖”，而应包含：

- 当前市场判断。
- 建议仓位范围。
- 适合资产类型。
- 主要风险。
- 触发加仓或减仓的条件。
- 模型置信度。
- 历史回测表现。

## 10. WebUI 预研结论

WebUI 应是工作台，不是宣传页。

一期页面：

- 首页仪表盘：市场状态、风险等级、今日建议。
- 数据页：资产行情、基金净值、指标曲线。
- 基金页：偏股型基金筛选、指标排名、评分。
- 预测页：模型输出、概率、收益区间、下行风险。
- 回测页：模型表现、历史预测评分、基准对比。
- 每日建议页：激进、中等、保守三类建议。
- 任务日志页：每日任务状态、错误和耗时。

## 11. MVP 实现建议

推荐一期实现顺序：

```text
1. 建立 Python 项目结构和 SQLite schema
2. 接入 AKShare，拉取 A 股指数、ETF、公募基金数据
3. 实现本地缓存和增量更新
4. 计算基础收益、波动、回撤、夏普等指标
5. 实现简单基准预测和评分
6. 实现历史滚动回测
7. 封装 MCP 工具
8. 建立每日 Codex 定时任务
9. 实现 WebUI 查看数据和建议
10. 引入机器学习模型并用回测评分替换或增强基准模型
```

## 12. 风险与合规边界

系统需要持续提示：

- 预测不是收益承诺。
- 模型可能失效。
- 历史表现不代表未来收益。
- 极端市场中相关性和流动性可能恶化。
- 建议必须结合用户风险承受能力和资金期限。
- 不应输出“稳赚”“保本”“确定收益”等结论。

## 13. 初步结论

一期 MVP 最合适的路线是：

```text
AKShare 数据接入
  + SQLite 本地持久化
  + 基础量化指标
  + 简单预测基线
  + 历史滚动回测
  + MCP 工具输出
  + Codex 每日 8 点分析任务
  + WebUI 展示结果
```

在这个基础上，再逐步加入 Tushare Pro、基金持仓、宏观数据、机器学习模型、多模型集成和组合优化。
